# 04. 常用命令与交互方式

Claude Code 的核心交互方式是自然语言加工具调用。你在终端中提出目标，Claude Code 根据需要读取文件、编辑文件、运行命令或调用外部工具。

## 启动

在项目目录运行：

```bash
claude
```

建议在仓库根目录启动。

## 常见自然语言指令

### 代码阅读

```text
解释这个项目的目录结构。
```

```text
找出支付流程的入口和关键文件，不要修改代码。
```

```text
这个函数在哪里被调用？
```

### 修改代码

```text
修复这个测试失败，保持改动最小。
```

```text
给用户设置页添加邮箱格式校验，并补充测试。
```

### 测试验证

```text
运行相关单元测试。
```

```text
启动应用并手动验证登录流程。
```

```text
检查当前改动是否通过类型检查和 lint。
```

### Git 工作流

```text
查看当前 git 状态和 diff。
```

```text
为当前改动创建提交，不要 push。
```

```text
创建 PR，描述包含 Summary 和 Test plan。
```

### Review

```text
审查当前 diff，重点关注正确性 bug。
```

```text
做一次安全审查，重点关注注入、权限和敏感信息泄漏。
```

## Slash commands 与 skills

Claude Code 支持以 `/` 开头的命令或技能。可用项取决于当前环境和安装配置。

常见类型包括：

- `/help`：查看帮助
- `/config`：调整配置
- `/review`：审查 PR
- `/security-review`：安全审查
- `/init`：初始化 CLAUDE.md
- `/loop`：定期执行任务

如果忘记某个命令，可以问：

```text
我想配置快捷键，应该用什么命令？
```

## Shell 命令

在 Claude Code 会话中，如果需要你自己运行 shell 命令，通常可以使用 `!` 前缀。例如：

```text
! npm test
```

这会在当前会话里运行命令，并把输出带回对话。

## 多轮协作技巧

### 先限制范围

```text
先只调查，不要修改。
```

### 要求列出文件

```text
列出你认为需要修改的文件和理由。
```

### 要求最小变更

```text
只修复这个 bug，不做无关重构。
```

### 要求验证

```text
修改后运行最小必要测试，并说明测试覆盖了什么。
```

### 要求风险说明

```text
总结剩余风险和我需要手动检查的地方。
```

## 长输出处理

当输出很长时，可以要求 Claude Code 摘要：

```text
运行测试，如果失败，只总结失败用例和关键错误，不要贴完整日志。
```

## 中断和改方向

如果发现方向不对，直接说：

```text
停。不要继续这个方向。回到只读状态，重新分析根因。
```

或者：

```text
撤销刚才对 X 文件的修改，保留 Y 文件。
```

涉及撤销文件时要谨慎，因为可能影响已有未提交改动。

## 好指令模板

```text
目标：修复 [问题]
现象：[实际行为]
期望：[期望行为]
范围：[允许修改的模块]
限制：[不要做什么]
验证：[需要运行的测试或手动检查]
输出：[希望最后总结什么]
```
