# 17. 常见问题与排错

本章整理 Claude Code 使用中的常见问题和排查思路。

## 命令找不到

现象：

```text
claude: command not found
```

检查：

- 是否已安装 Claude Code
- 是否重启终端
- PATH 是否正确
- 当前 shell 配置是否加载

## 登录失败

可能原因：

- 浏览器无法打开
- 网络代理问题
- 账号权限问题
- 远程环境无法完成回调

建议：

- 按 CLI 提示重新登录
- 检查网络和代理
- 在本机而非远程环境完成认证

## Claude Code 找不到项目

检查是否在仓库根目录启动。

可问：

```text
查看当前目录、git 状态和项目文件，判断我是否在正确目录。
```

## 权限提示太多

原因可能是：

- 默认权限较严格
- 频繁运行 shell 命令
- 没有配置只读命令允许列表

解决：

- 允许常用只读命令
- 配置项目级 settings
- 使用减少权限提示的 skill

但不要为了省事允许危险命令。

## Claude Code 修改方向不对

立即中断并重设范围：

```text
停。不要继续修改。请总结你已经改了什么，然后回到只读分析。
```

如果需要撤销，要先检查 diff，避免覆盖用户其他改动。

## 测试命令失败

先判断：

- 是本次修改导致吗
- 是环境缺依赖吗
- 是已有失败吗
- 是命令用错了吗

提示词：

```text
分析测试失败是否与本次修改相关，不要跳过测试。
```

## 依赖安装失败

检查：

- 包管理器是否正确
- lockfile 是否存在
- Node/Python/Go 版本
- 私有 registry 权限
- 网络代理

不要随意删除 lockfile 或降级依赖。

## Git push 失败

可能原因：

- 没有 remote
- 没有权限
- 分支没有 upstream
- 远程有新提交
- token 缺少权限

建议先：

```text
查看 remote、当前分支和 git status，分析 push 失败原因。
```

## PR 创建失败

可能原因：

- 未安装 GitHub CLI
- 未登录
- token 权限不足
- 当前分支未推送
- 没有 GitHub remote

可以改用 GitHub API，但必须确保 token 存在且权限足够。

## Hooks 阻止操作

不要立刻绕过。先阅读 hook 输出，判断是否触发了安全规则。

如果误报，再调整 hook 配置。

## MCP 连接失败

检查：

- MCP server 是否启动
- 配置路径是否正确
- token 是否存在
- 网络是否可达
- 工具名称是否变化

## 输出太长

要求 Claude Code 限制输出：

```text
只总结关键错误和下一步，不要贴完整日志。
```

## 上下文混乱

如果长对话后 Claude Code 开始混淆，可以：

- 要它重新读取当前文件
- 开新会话
- 把稳定规则写入 CLAUDE.md
- 不把临时状态写入 memory

## 误提交敏感信息

如果还没 push：

1. 从提交中移除
2. 重新提交
3. 轮换密钥

如果已经 push：

1. 立即轮换密钥
2. 通知团队
3. 按组织流程清理历史
4. 审计访问日志

不要只删除文件就认为安全。
