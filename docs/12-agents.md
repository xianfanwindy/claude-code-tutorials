# 12. Agents 与子代理

Agents 是 Claude Code 中用于分工的子代理。它们适合处理相对独立的调查、搜索、审查或专门任务。

## 为什么需要子代理

大型项目中，单个上下文可能很快被大量文件和日志填满。子代理可以：

- 并行探索不同模块
- 隔离大量搜索结果
- 让专门代理处理专门任务
- 减少主对话上下文压力

## 适合使用 Agent 的场景

### 大范围代码搜索

```text
找出所有处理用户权限的模块，并总结入口。
```

### 独立审查

```text
请给这个数据库迁移做第二意见审查。
```

### 并行调查

一个代理查前端，一个代理查后端，最后由主对话综合。

### 专门领域任务

例如 Claude Code 用法问题、API 开发、状态栏配置等。

## 不适合使用 Agent 的场景

- 已经知道文件路径的简单读取
- 单行修复
- 小范围 grep
- 需要主对话直接控制的精细编辑

## 常见代理类型

### Explore

快速只读搜索代理，适合定位文件、符号和引用。

### Plan

用于复杂实现前制定计划。

### claude-code-guide

用于回答 Claude Code、Claude API、Agent SDK 等相关问题。

### general-purpose

通用代理，适合复杂研究或多步骤任务。

## 使用原则

### 给足上下文

子代理看不到主对话的全部细节。调用时要说明：

- 目标
- 背景
- 已知信息
- 输出格式
- 是否允许修改

### 不要把理解外包

主对话应该负责综合判断。子代理提供发现，不应直接替代主判断。

### 结果要验证

如果子代理修改了代码，主对话应检查实际 diff 和测试结果。

## 示例：并行调查

```text
让一个代理调查前端登录状态，另一个代理调查后端 session API，最后汇总根因。
```

## 示例：代码定位

```text
使用 Explore 代理快速查找 uploadFile 函数定义和所有调用点。
```

## 输出要求

对子代理设置输出限制很重要：

```text
请在 200 字内报告关键文件、根因猜测和下一步建议。
```

否则它可能返回过长内容。

## 风险

- 子代理可能漏读文件
- 子代理可能误解背景
- 多代理结果可能冲突
- 修改型代理需要额外审查

因此主对话需要“信任但验证”。
